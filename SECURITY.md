# Security Policy

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

We take the security of our project seriously. If you find a security vulnerability, please do **not** open a public issue. Instead, please report it via one of the following methods:

- **Email**: [INSERT SECURITY EMAIL]
- **GitHub**: Open a draft security advisory through the GitHub repository (if available).

Please include the following details in your report:

- The type of issue (e.g., buffer overflow, SQL injection, cross-site scripting).
- A description of the vulnerability.
- Steps to reproduce the issue.
- Possible impact of the vulnerability.

We will acknowledge receipt of your report and provide a timeline for addressing the issue. We ask that you follow responsible disclosure practices and give us time to fix the vulnerability before making it public.
